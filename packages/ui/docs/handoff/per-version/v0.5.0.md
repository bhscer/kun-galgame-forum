# v0.5.0 handoff — KunImage 加载骨架屏

## 新功能

`<KunImage>` 加载时自动显示 pulse 骨架屏（`bg-default-200 animate-pulse`），加载完成 / 失败时自动移除。SSR 阶段就有 skeleton class 直出 HTML，hydration 后无缝接管。

## 同步 — 1 个文件

```bash
KUN_OAUTH=/path/to/kun-oauth-admin
cp $KUN_OAUTH/packages/ui/app/components/kun/image/Image.vue \
   ./components/kun/image/Image.vue
```

## 消费侧

**零修改即享受**。所有 `<KunImage>` 调用点自动有 skeleton。

如果某个场景不想要 skeleton（比如装饰用的 32x32 小 icon、pulse 反而 distracting）：

```vue
<KunImage :src="iconSrc" :skeleton="false" width="32" height="32" />
```

## 设计取舍

我没用"包 wrapper div" 的常见做法 —— 那是**破坏性改动**（KunImage 渲染出 `<div><img></div>` 而不是单个 `<img>`，layout 假设全乱）。改用 **同一个 `<img>` 上加 `bg-default-200 animate-pulse` 背景类**，加载完移除。

| 维度 | wrapper 方案 | bg-class 方案（采纳） |
|---|---|---|
| 渲染元素类型 | `<div><img></div>` | 单个 `<img>` |
| 调用方兼容性 | ❌ 破坏 | ✅ 零破坏 |

## 必须搭配 `width` + `height`

`<img>` 没尺寸 = 0×0 = skeleton 不可见。这跟 v0.4.6 文档已经在说的 "防 layout shift 必传 width/height" 是同一前提：

```vue
<!-- 推荐 -->
<KunImage :src="banner" :width="512" :height="288" />

<!-- 也行（父级有 aspect ratio + 自身 size-full） -->
<div class="aspect-video">
  <KunImage :src="banner" class-name="size-full object-cover" />
</div>
```

## 三段加载视觉链

如果同时传 `placeholder="true"`（NuxtImg 内置低质量模糊预览）：

```
[skeleton pulse 灰色]  →  [模糊 placeholder]  →  [清晰 final image]
```

`@load` 在最终图 ready 时触发，所以 skeleton cover 到模糊预览阶段，视觉无缝。
