# v0.6.1 handoff — KunContent 内置图片 lightbox + 收口 useKunLightbox drift

本版把"markdown 图片点击放大"沉进 `KunContent`。对下游（moyu / kungal）有一个**特殊收口动作**：你们之前可能自行在本地 fork 重写过 `useKunLightbox`（事件委托版），现在 canonical 出了正式方案，需要删本地副本回归跟随 upstream。

## 同步文件

```bash
KUN_OAUTH=/path/to/kun-oauth-admin
DEST=/path/to/your-app   # moyu 或 kungal 仓库根

# 1. 新增 composable
cp "$KUN_OAUTH/packages/ui/app/composables/topic/useContentLightbox.ts" \
   "$DEST/packages/ui/app/composables/topic/"

# 2. 覆盖 KunContent（内置了 lightbox + img cursor）
cp "$KUN_OAUTH/packages/ui/app/components/kun/content/Content.vue" \
   "$DEST/packages/ui/app/components/kun/content/"

# 3. 删除老的 broken useKunLightbox（canonical 已删）
rm -f "$DEST/packages/ui/app/composables/useKunLightbox.ts"

# 4. ⚠️ 关键：删除你们 APP 层（apps/web）可能存在的本地副本
#    （app 层 > layer，会覆盖 KunUI 版导致 drift —— 这就是之前踩的坑）
rm -f "$DEST/apps/web/app/composables/useKunLightbox.ts"
```

## 收口背景（为什么要删两个 useKunLightbox）

历史上 moyu 在 `apps/web/app/composables/useKunLightbox.ts` 放了本地副本，**app 层优先级 > extended layer**，把 KunUI 的 layer 版盖掉了。两者各自演化 → drift。

canonical 现在的方案是 `useContentLightbox` + KunContent 内置，**不再有独立 `useKunLightbox`**。所以：
- KunUI 的 `useKunLightbox.ts` 已删
- 你们 app 的本地副本也要删
- 凡是直接 `useKunLightbox()` 的调用点，改用下面任一方案

## 消费方迁移：两条路

### 路线 A（推荐）：手搓 prose → `<KunContent>`

如果你的评论/正文是手搓的：

```vue
<!-- 旧 -->
<div class="kun-prose" v-html="sanitize(c.content_html)" />

<!-- 新 -->
<KunContent :content="c.content_html" />
```

`<KunContent>` 自带 sanitize + spoiler + 图片 lightbox，**全部白嫖，零额外 wiring**。注意去掉自己那层 DOMPurify（KunContent 内部已经做了）。

> 评论卡片整卡可点跳转的场景（comment/Card.vue）：KunContent 内部的 lightbox click 已 `stopPropagation`，点图片不会触发卡片导航。但 `<KunContent>` 要放在卡片可点区域内即可，无需特殊处理。

### 路线 B（escape-hatch）：保留手搓 div，但用 composable

如果某处暂时不能迁到 KunContent（比如有特殊 sanitize 配置）：

```vue
<script setup>
const proseRef = ref<HTMLElement | null>(null)
const { isLightboxOpen, images, currentImageIndex } = useContentLightbox(proseRef)
</script>

<template>
  <div ref="proseRef" class="kun-prose" v-html="sanitize(html)" />
  <KunLightbox v-model:is-open="isLightboxOpen" :images :initial-index="currentImageIndex" />
</template>
```

注意：`useContentLightbox` 需要传你自己的容器 ref。

## 验证

```bash
cd /path/to/your-app
pnpm prepare              # 不报 dup composable warning
pnpm typecheck            # 通过（确认没有残留 useKunLightbox 引用）
pnpm -F your-app exec nuxt build
```

grep 确认无残留：
```bash
grep -rn 'useKunLightbox' apps/ packages/ | grep -v '\.nuxt'
# 应为空 —— 全部迁到 useContentLightbox 或 KunContent
```

## 视觉手测 checklist

- [ ] 评论里的图片：点击弹 lightbox，显示该图
- [ ] 多图评论：lightbox ←/→ 在**该评论自己的图**之间切换（不跨评论）
- [ ] 翻页/加载更多评论后：**新评论里的图片也能点开**（这是事件委托的核心收益）
- [ ] 评论卡片可点跳转的列表：点图只开 lightbox，不触发卡片导航
- [ ] spoiler 里的图：先 reveal spoiler，再点图开 lightbox（两交互不打架）
- [ ] dark mode：lightbox 工具栏清晰（v0.6.0 已修）
