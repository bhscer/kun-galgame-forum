# v0.5.1 — KunDrawer：边缘锚定的浮层

## 跟 KunModal 的边界

| 维度 | KunModal | KunDrawer |
|---|---|---|
| 锚定 | 视口中心 | 视口某一边（left / right / top / bottom）|
| 关闭方式 | backdrop 点击 / Escape / 关闭按钮 | 同 |
| 阻塞性 | 全屏背景遮罩 + 焦点陷阱 | 同 |
| 内容尺寸 | 由内容决定（max-h 90vh） | 由 `size` prop 决定（cross-axis 固定，main-axis 占满） |
| 滚动锁 | ✅ useBodyScrollLock | ✅ useBodyScrollLock |
| 适用场景 | 表单弹窗、确认对话框、详情查看 | 侧边导航、筛选面板、长内容详情、设置面板、购物车 / 通知抽屉 |
| z-token | `z-kun-modal` | `z-kun-modal`（同层，drawer 嵌套 modal 不需要） |

**何时用 Modal vs Drawer 的简单规则**：

- 用户要**专注做一件事**（确认 / 表单填写）→ Modal
- 用户要**浏览补充内容**（看列表 / 筛选 / 看长详情）→ Drawer

## 实现要点

1. **同源 mechanics 共用** —— useBodyScrollLock / useFocusTrap / useEventListener Escape / Teleport to body / v-model 全部抄 Modal 那一套，**消费方零学习成本**
2. **placement × size 矩阵** —— `placement: left/right/top/bottom` × `size: sm/md/lg/xl/full`
   - left/right size 控制 width（80px → 640px → 100%）
   - top/bottom size 控制 height（同尺度）
3. **slide 动画** —— 每个 placement 一套 `kun-drawer-{placement}` Transition：
   - left: `translateX(-100%)` → 0
   - right: `translateX(100%)` → 0
   - top: `translateY(-100%)` → 0
   - bottom: `translateY(100%)` → 0
   - backdrop 同步淡入淡出（opacity transition）
4. **圆角只露内侧** —— 抽屉贴边的那条边保持直角，朝内的两个角圆角。`placementRoundedSideMap` 用 Tailwind `rounded-{l/r/t/b}-kun-lg` 实现。`size: full` 跳过所有圆角（贴满对边时圆角看着怪）
5. **slot 三段** —— `header` / 默认 / `footer`，分别 `border-b` / 中部 `flex-1 overflow-y-auto` / `border-t`，符合"app-like 抽屉"的标准布局

## 关键 prop 列表

```ts
interface KunDrawerProps {
  placement?: 'left' | 'right' | 'top' | 'bottom'   // 默认 'right'
  responsive?: boolean                               // 默认 true（手机自动转底部）
  size?: 'sm' | 'md' | 'lg' | 'xl' | 'full'         // 默认 'md'
  title?: string                                     // 可选标题，渲染进 header
  isDismissable?: boolean                            // 默认 true，背景点 + Escape 关
  isShowCloseButton?: boolean                        // 默认 true，header 右上 X
  withContainer?: boolean                            // 默认 true，body 用 p-6 + overflow-y-auto 包
  rounded?: KunUIRounded                             // 默认 'lg'，仅作用于内侧两个角
  className?: string                                 // 透到外层 overlay
  innerClassName?: string                            // 透到 panel div
}
```

## 用法示例

### A. 标准侧边抽屉（最常见用法）

```vue
<script setup>
const open = ref(false)
</script>
<template>
  <KunButton @click="open = true">打开侧栏</KunButton>
  <KunDrawer
    v-model="open"
    placement="right"
    size="md"
    title="详细筛选"
  >
    <p>抽屉正文区...</p>
    <template #footer>
      <KunButton variant="solid" color="primary" @click="open = false">
        应用筛选
      </KunButton>
    </template>
  </KunDrawer>
</template>
```

### B. 移动端导航 drawer（左侧 + full size，覆盖整屏）

```vue
<KunDrawer
  v-model="navOpen"
  placement="left"
  size="full"
  title="导航"
>
  <NavMenu />
</KunDrawer>
```

### C. 自定义 header（不用 title prop）

```vue
<KunDrawer v-model="open" placement="right">
  <template #header>
    <div class="flex items-center gap-2">
      <KunAvatar :user="user" size="sm" />
      <h2 class="text-lg font-semibold">{{ user.name }}</h2>
      <KunChip color="success">在线</KunChip>
    </div>
  </template>
  ...
</KunDrawer>
```

### D. 非阻塞性（只想用 X 关，不让背景点关）

```vue
<KunDrawer v-model="open" :is-dismissable="false">
  ...
</KunDrawer>
```

适合"用户正在编辑表单，不能因为误点背景丢数据"的场景。

## 实现 cheat sheet

**panelGeometry** computed 把 placement + size 翻译成 anchor classes：

```ts
// placement="right" + size="md" 的 panel:
'absolute right-0 top-0 h-full w-96 rounded-l-kun-lg'
//                  ↑ anchor ↑ axis ↑ size ↑ inner-side rounded

// placement="bottom" + size="lg":
'absolute bottom-0 left-0 w-full h-[32rem] rounded-t-kun-lg'
```

**Transition 名字动态切**：`transitionName = "kun-drawer-${placement}"`。4 套 CSS 各负责一条边的 slide 方向。背景 fade 用一个共同的 `opacity 0.3s ease` 规则。

## a11y

- `role="dialog"` + `aria-modal="true"`
- `aria-labelledby="<uniqueId>-title"`（如果传了 title）
- Escape 关闭（通过 `useEventListener('keydown')`，受 `isDismissable` 控制）
- 焦点陷阱：Tab/Shift+Tab 不会跑出 drawer
- `returnFocusOnDeactivate: true`：关闭后焦点回到打开前的元素
