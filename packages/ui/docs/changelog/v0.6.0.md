# v0.6.0 — 图片查看器重写 + KunTooltip 精简 + glass-blur token 修复

跨多个组件的批次。主线是 **KunLightbox 重写 + 新增 KunLightboxGallery/Item 声明式包装**（图片画廊到查看器的现代 Vue 模式），副线是几个长期欠的修复（KunTooltip 误触发 / `--kun-background-blur` CSS 变量缺失 / Card.isTransparent 实际无效）。

## §1 新增：`--kun-background-blur` CSS 变量

`packages/ui/app/styles/tailwindcss.css` 的 `:root` 块加：

```css
--kun-background-blur: 0px;
```

### 为什么是个 bug

`KunCard` (`Card.vue:92`) 和 `KunModal` (`Modal.vue:126`) 都用了 `backdrop-blur-[var(--kun-background-blur)]`，但这个变量**整个 oauth-admin 项目从未定义过**。grep 仓库零结果，连 JS 也没人 `setProperty`。

对比 kungal 老项目（`kun-galgame-nuxt4/apps/web/app/store/modules/settings.ts:52`）有完整设置链路（Pinia store + 设置面板）写 `document.documentElement.style.setProperty('--kun-background-blur', '${blur}px')`。oauth-admin 把组件搬过来但没搬这套设置 → 变量永远未定义。

CSS 行为：`backdrop-filter: blur(var(--kun-background-blur))` 解析为 `blur()`（空参数）= **invalid value → 整条 backdrop-filter 失效**。Card / Modal 的玻璃磨砂效果**从未真正工作过**。

### 修法：默认 `0px`

最小侵入：把变量定义在 `:root`，默认 0px = 无模糊，匹配 kungal-legacy 的 Pinia 默认。下游如想开启磨砂效果，可在 layout / settings store 写：

```ts
document.documentElement.style.setProperty('--kun-background-blur', '12px')
```

立刻给 Card / Modal / 任何引用同 token 的组件加上磨砂效果，**零组件改动**。

### 副作用：`KunCard.isTransparent` 现在有可见差异

`isTransparent` prop 的设计本意是切换"含玻璃磨砂的 chrome 卡片"vs"完全透明卡片"。但因为 var 未定义，两态视觉等同（都没磨砂）。修了 var 之后这个 prop 才有了实质区别。Card 代码本身**未改**。

## §3 KunLightbox — 全面重写

`packages/ui/app/components/kun/lightbox/Lightbox.vue`：从 405 行 → 880 行（增长主因：手势状态机 / 截图布局 / 完整注释）。

完整组件文档见 [`components/kun-lightbox.md`](../components/kun-lightbox.md)。这里仅列变更摘要。

### 3.1 弹层容器：`KunModal` → 原生 `<dialog>`

- 免费拿到：focus trap、ESC 关闭、background `inert`、`::backdrop` 伪元素
- 仍手动管：`useBodyScrollLock`（dialog 原生不锁 body 滚动）

### 3.2 切图：carousel 滑动动画

- 之前：直接 src 替换，无过渡
- 现在：Vue `<Transition :name="slideDir">` + 绝对定位重叠 + 350ms cubic-bezier
- 方向感知：next → 旧图飞左、新图从右进；prev 反之

CSS（新增 4 个类）：
```css
.slide-next-enter-from { transform: translateX(100%); opacity: 0; }
.slide-next-leave-to { transform: translateX(-100%); opacity: 0; }
/* slide-prev 镜像 */
```

### 3.3 视图过渡：`document.startViewTransition` 集成

```ts
const withViewTransition = (mutate: () => void) => {
  const doc = document as Document & { startViewTransition?: (cb: () => void) => unknown }
  if (typeof doc.startViewTransition === 'function') doc.startViewTransition(mutate)
  else mutate()
}
```

打开 / 关闭 / 翻页都包了 `withViewTransition(() => ...)`。支持浏览器（Chrome/Edge 111+、Safari 18+）自动 cross-fade；不支持的降级到直跑。

进阶能力：consumer 给缩略图加 `view-transition-name: kun-lightbox-image` CSS，浏览器自动计算 morph 从缩略图到全屏图的轨迹（默认 fade）。

### 3.4 旋转：左右两键 + 90° 步进

- 左旋（`lucide:rotate-ccw`）：`rotation -= 90`
- 右旋（`lucide:rotate-cw`）：`rotation += 90`

`rotation` **存储为无界数值**（±∞），不取模。原因：CSS `transition: transform` 按数值插值。如果取模，在 wrap 边界（如 270° → 0°）会插值 270° 反方向 = 旋转 270° 逆时针，视觉眩晕。无界存储确保插值永远朝用户点击的方向走。

重置时智能 snap：
```ts
rotation.value = Math.round(rotation.value / 360) * 360
```
吸附到最近的 360 倍数（视觉等同 0°），动画路径 ≤180°，避免 0.3s 内 unwind N 圈的眩晕。

### 3.5 工具条：底部居中 + 多区段分组

旧：四角散落（prev/next 左右、close 右上、download 右下）
新（参考用户提供的 PC + mobile 截图）：

```
顶部：[1/4] ────────────── [×]   (counter / close)
中部：[‹]    image    [›]         (PC 大箭头，mobile 隐藏)
底部 mobile：[● ○ ○ ○]            (pagination dots)
底部 PC    ：[□][□][□][□]         (thumbnail strip，max-w-[80vw] 横滚)
底部全部   ：[− 100% +] [↺ ↻] [⟲ ⬇]  (zoom / rotate / reset+download)
```

各 chrome 都是独立 pill，外层 `pointer-events-none` 包裹，每个 pill `pointer-events-auto` —— 这样 pill 之间的空隙仍能点击 backdrop 关闭，不变成隐形屏障。

### 3.6 Dark mode 配色修复

旧：`bg-default-900/70 text-white border-default-200/30`
新：`bg-black/70 text-white border-white/10`

原因：`--default-900` 自动反转色阶 → dark mode 下变成浅色 → 白底白字。所有图片查看器（YouTube / Photoshop / Lightroom）的工具栏惯例都是**永远深色玻璃**，不随主题反转。

### 3.7 下载 CORS fallback

旧：`fetch → blob → <a download>`，CDN（Cloudflare R2 默认无 CORS 头）跨域 → `TypeError: Failed to fetch` 直接报错。

新：try-catch + 失败 fallback `window.open(url, '_blank')` 让用户右键保存。console 降到 warn 而非 error（这是 expected recovery）。长期方案：让运维给 R2 加 CORS rule。

### 3.8 手势 bug 全清

1. **手机端双击不工作** — `@dblclick` 在 touch 上不可靠（iOS Safari 经常不合成）。加手动 double-tap 检测：300ms 间隔 + 30px 位移阈值。
2. **pinch 缩放有 0.3s lag**（"动画僵直"根因）— 新增 `isPinching` ref。`transformStyle.transition` 在 `isDragging || isPinching` 时设 `none`，pinch 帧对帧跟手。
3. **pinch ↔ pan baseline 不重置** — 双指放开变单指时，旧代码不重置 `dragStart`，单指 pan 第一帧瞬移。修：`onTouchEnd` 检测从 pinch 退化到 1 指，重新 `startDrag`-style 初始化 baseline。
4. **pinch zoom 公式错** —— 旧 `(position + dx) * scaleChange` 不是标准 anchored zoom，图片"漂"过手指方向。改用 wheel-zoom 同款公式：`anchor - (anchor - oldPos) * scaleChange`，anchor 是两指中点相对容器中心。
5. **浏览器原生 pinch-zoom 抢手势** — container 加 `touch-none`（= `touch-action: none`），JS 完全接管图片区域所有触摸。
6. **`* 0.01` magic number** — 改为 `currentDist / lastDist` 距离比例，scale-aware，低 zoom 不再跳变。

## §4 新增：`KunLightboxGallery` + `KunLightboxGalleryItem`

声明式画廊包装。Pattern 2（vs Pattern 1 = 显式数组）。

文件：
- `packages/ui/app/components/kun/lightbox/Gallery.vue`（107 行）
- `packages/ui/app/components/kun/lightbox/GalleryItem.vue`（98 行）

实现：Item 用 `provide/inject` 注册到 Gallery 的 reactive items[]，Item 点击 → Gallery 打开内置单实例 `<KunLightbox>` 定位到该 index。

API 详见 [`components/kun-lightbox.md`](../components/kun-lightbox.md) §Gallery 段。

### 关键设计点

1. **InjectionKey 在 Gallery.vue 内导出**（非 setup `<script>` 块），而非独立 `.ts` 文件 — 否则 Nuxt 把它当成 component 注册（`KunLightboxGalleryKey` 幽灵组件）
2. **Item 包装是 `<span class="cursor-zoom-in">`** + `role="button" tabindex="0"` + 键盘 enter/space — a11y 完备
3. **scoped-slot 暴露 `{ open }`** — 进阶用法可 `:wrap="false"` 关 wrapper，自己控制触发位置
4. **多个 Gallery 同页互不干扰** — `provide` 是 instance-scoped
