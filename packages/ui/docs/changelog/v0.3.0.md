# v0.3.0 — 统一 rounded 设计 token 系统（2026-05-21）

主题：给所有 stylistic 组件加一个统一的圆角调整入口，三层（CSS token / Vue provider / 组件 prop）按优先级覆盖。

详细设计文档单独留在了 [`../architecture/rounded-system.md`](../architecture/rounded-system.md) —— 它讲清楚了：

- **Layer 1（最底层）**：`packages/ui/app/styles/tailwindcss.css` 的 `@theme` 块定义 5 个 `--radius-kun-*` token（`none / sm / md / lg / full`），Tailwind v4 自动生成 `rounded-kun-*` utility
- **Layer 2（Vue 注入）**：`useKunUIConfig` provider 让消费方在 app.vue / 顶层 layout 一次性设全局默认（如 `provideKunUIConfig({ rounded: 'lg' })`），或者用 KunUIScope 包裹局部子树覆盖
- **Layer 3（组件实例 prop）**：每个组件接受 `rounded` prop，最细粒度

**优先级**：组件 prop > useKunUIConfig 提供值 > 组件内置 fallback > 全局默认 `'md'`

## 13 个 stylistic 组件全部接入

KunCard / KunButton / KunModal / KunInput / KunTextarea / KunSelect / KunPopover / KunTooltip / KunDatePicker / KunTagInput / KunUpload / KunProgress / KunInfo

具体每个组件的 fallback 默认值（Modal/Popover/Upload/Info 用 `lg` built-in，其他默认 `md`）见 [`../architecture/rounded-system.md`](../architecture/rounded-system.md) 的"已接入的组件"表。

## 形状本质组件不参与

KunAvatar / KunChip / KunBadge dot-count / Switch thumb / Slider thumb / KunTab pills variant —— 这些组件的圆角是**语义**而非样式选择，**忽略 useKunUIConfig**。即使全局设 `rounded: 'none'` 它们也保持自己的圆形 / 胶囊形。

## 兼容性

完全非破坏性。所有现有调用零修改，视觉零差异（fallback 默认 `md` 对应 0.5rem = 原硬编码值）。
