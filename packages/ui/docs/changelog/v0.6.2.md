# v0.6.2 — 新增 KunDropdown（click 触发的操作菜单 / WAI-ARIA menu button）

补齐 KunUI 缺失的"点击按钮 → 弹出一组可点操作项"组件。此前最接近的 `KunPopover`（click）只是通用浮层，`KunContextMenu` 是右键/点位菜单，`KunTooltip` 是 hover 文字提示 —— 都不是带 `role="menu"` 语义 + 键盘导航的操作菜单。

## 为什么是独立组件，而不是给 KunPopover 加 `trigger`

评估过"复用 KunPopover"路线，结论是 **Popover 太封闭，无法 compose 出合规的 menu**：

1. **内容容器 role 写死 `role="dialog"`**（Popover.vue），菜单需要 `role="menu"` + `menuitem`，改不了。
2. **open 状态是内部私有的** —— Popover 不 emit `update:open`、不暴露响应式 `isOpen`，且自己处理 Esc / outside-click 关闭。menu 需要据此驱动 `aria-expanded`、open 时聚焦首项、close 时焦点归位 —— compose 时这些状态会和 Popover 内部脱同步。

所以 KunDropdown **直接 wrap `@floating-ui/vue`**（与 Popover 同一套定位配方：`useFloating + autoUpdate + offset/flip/shift`、`transform:false`、Teleport、Transition），自带交互 + a11y 层；**不动 Popover、不抽公共基座**（YAGNI —— 暂时没有第三个浮层组件来指导抽象）。item 数据模型直接复用 `KunContextMenuItem`，视觉复用共享 `kunVariantClasses('light', color)` 表。

## 为什么没有 hover 触发

刻意不做。hover 菜单不是 WAI-ARIA 模式（触屏无 hover、键盘/读屏用户够不着），且本组件的用例是"一组可点的操作项"。需要的就是标准 menu button：**click / Enter / Space / ↓ 打开**。

## API

```vue
<KunDropdown :items="items" @select="onSelect">
  <template #trigger>
    <KunButton variant="flat">操作菜单</KunButton>
  </template>
</KunDropdown>
```

| Prop | 类型 | 默认 | 说明 |
|---|---|---|---|
| `items` | `KunDropdownItem[]` | `[]` | 菜单项（= `KunContextMenuItem`：`key / label / icon? / color? / disabled?`）|
| `position` | `Placement` | `'bottom-start'` | floating-ui 锚定方向，自动 flip/shift 进视口 |
| `triggerClass` | `string` | `''` | 触发器外层 class |
| `menuClass` | `string` | `''` | 菜单容器 class |
| `minWidth` | `number` | `192` | 菜单最小宽度（px）|
| `disabled` | `boolean` | `false` | 禁用整个触发器 |

Emits：`select(item)`、`open()`、`close()`。Expose：`open() / close() / toggle()`。

## a11y / 键盘（WAI-ARIA menu button）

| 操作 | 行为 |
|---|---|
| Click / Enter / Space / ↓ | 打开（↓/Enter/Space 同时聚焦首项）|
| ↑（在触发器上）| 打开并聚焦末项 |
| ↑ / ↓ | 在启用项间移动（roving tabindex，循环，**跳过 disabled**）|
| Home / End | 跳到首 / 末启用项 |
| Enter / Space（项上）| 触发 `select` 并关闭，焦点归位触发器 |
| Esc / Tab | 关闭并焦点归位触发器 |
| 点击外部 | 关闭 |

触发器是 `div[role=button]` + `aria-haspopup="menu"` + `aria-expanded`；菜单是 `role="menu"` / 项是 `role="menuitem"`。light 变体只有 `hover:`，键盘焦点不触发 hover，故为每个语义色补了 `focus:` 高亮，保证键盘 roving 与鼠标 hover 视觉一致。

> **触发器约定**：与 `KunPopover` 一致 —— 把内容（或 `<KunButton>`）放进 `#trigger`，组件用 `div[role=button]` 包裹（不是真 `<button>`，避免 button 套 button）。

## 验证

- `nuxt prepare` + `vue-tsc --noEmit`：零报错；auto-import 注册为 `KunDropdown`。
- Playwright 实测（wiki dev server）：click 打开 → ↓↓↓ 跳过 disabled "归档" → Enter 选中 "删除" 触发 select + 关闭 + 焦点归位 → Esc 关闭归位；danger 红 / disabled 置灰 / 图标 / bottom-start 定位均正确，走项目色系。
- 注：开发期编辑 layer（packages/ui）文件后，apps 的 Vite 客户端 bundle 可能短暂缓存旧版（SSR 用新版 → 出现一次性 hydration mismatch 告警）；**重启 dev server 即清除**，非组件问题。
